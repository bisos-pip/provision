\section{Overview}

Bash scripts (Interactive Command Modules -- ICM) for provisionping
BISOS (ByStar Internet Services OS) software profiles on a virgin
Linux distro.
Or for creating fully automated KVM guests that are ByStar Platforms.

Running \textbf{bx2Genesis.sh} will go through a complete ByStar Platform installation on the local host.

Running \textbf{bxProvisionGuest.sh} will create a fully automated KVM Guest ByStar Platform.

\section{Installation}

\begin{verbatim}
sudo pip install bisos.provision
\end{verbatim}


\section{Usage}

\subsection{bx2Genesis.sh}

% \begin{verbatim}
% /usr/local/bin/bx2Genesis.sh
% /usr/local/bin/bxProvisionGuest.sh
% \end{verbatim}

On a virgin BxP-Distro, run ``sudo /usr/local/bin/bx2Genesis.sh'' and 
you will end up with a Generic BISOS Platform. Which you can then
apply to a desired bxpCharacter.

\subsection{bxProvisionGuest}

On any Linux machine that has kvm in its distro, run 
/usr/local/bin/bxProvisionGuest.sh and you will end up with 
a guest at the specified desired level (as ByStar Platforms).

\subsection{"./bin/bxHostGenGuestVagrant"}

On any Linux Machine that has VirtualBox and Vagrant installed,
run bxHostGenGuestVagrant and based on params and args build a VM
that includes what is specified.

The Steps are as follows:
\begin{itemize}
\item Create A VM
\item In the created VM as root:
  \begin{itemize}
  \item install python and pip
  \item install git
  \item pip install bisos.provision
  \item Run xxx to create user bxGenesis and add it to sudoers    
  \end{itemize}
\item In the created VM as bxGenesis run bisos.provision/bin/bxGenWithRepo
\item Login to the VM as bxGenesis
\item Run the post install script.  
\end{itemize}  

\subsection{"./bin/bxGenWithRepo"}

Does the following:

\begin{itemize}
\item Clone specified repo
\item From within that repo executes specified entry point with params and args. \
  This typically involves creating an account
\end{itemize}
